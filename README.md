# ЁЯУ╕ Google Photos тЖТ Google Drive Backup Tool

A toolkit to identify **original-quality photos and videos** that consume your Google storage quota and back them up into **year-wise folders** on Google Drive using `rclone`.

---

## ЁЯза The Problem

Google Photos stores two kinds of files:

| Type | Takes Storage? | Description |
|---|---|---|
| **Original Quality** | тЬЕ Yes | Uploaded in full resolution тАФ counts against your 15 GB quota |
| **Storage Saver** | тЬЕ Yes (post Jun 2021) | Compressed by Google тАФ counts against quota since June 2021 |
| **Storage Saver** | тЭМ No (pre Jun 2021) | Old compressed uploads тАФ were free, don't count |

**Goal**: Find only the files that are **Original Quality AND taking up space**, then make a clean backup of them in Google Drive organized by year.

### Why not use the Google Photos API?
- The official API **strips GPS/EXIF metadata** from downloaded files
- The API has **no filter for "Original Quality"** тАФ Google does not expose this as a queryable field
- The API has **no `takesUpSpace` field** тАФ storage quota status is server-side only

### Why not use Google Takeout?
- Takeout exports **everything** (all years, all files) тАФ takes hours to download
- Requires merging metadata from JSON sidecar files using third-party tools
- No way to pre-filter "only originals that take space" before downloading

### тЬЕ Our Approach
Use the **Google Photos Toolkit userscript** (which hooks into Google's internal API via your browser session) to get an accurate metadata CSV тАФ including `takesUpSpace` and `isOriginalQuality` тАФ then match those filenames against your existing Google Drive and copy them into organized folders, **without downloading anything to your computer**.

## ЁЯУж Files in This Project

| File | Purpose |
|---|---|
| `google_photos_toolkit.user.js` | Modified GPTK userscript тАФ adds an "Export JSON" button (optional) |
| `metadata.csv` | Output from GPTK's "Export Metadata" button тАФ your source of truth |
| `backup_photos.py` | Main script тАФ reads the CSV, matches Drive files, generates backup commands |
| `media_filter.txt` | rclone filter тАФ only indexes photo/video file types from Drive |
| `drive_index.json` | Generated by rclone тАФ a local index of all media in your Drive |
| `run_backup.sh` | Generated by `backup_photos.py` тАФ contains all the rclone copy commands |
| `missing_files.txt` | Generated if some files from the CSV weren't found in Drive |

## ЁЯЪА Step-by-Step Workflow

### Step 1 тАФ Get your file list from Google Photos

1. Install the modified `google_photos_toolkit.user.js` in Tampermonkey
2. Go to [photos.google.com](https://photos.google.com)
3. Open the GPTK panel (the wrench icon in the top bar)
4. In **Filters**:
   - **Space** тЖТ select `Consuming` (only files that use your storage)
   - **Quality** тЖТ select `Original` (only original quality, not Storage Saver)
5. In **Actions** тЖТ click **`Export Metadata`**
6. Save the downloaded file as `metadata.csv` in this project folder

> **What you get**: A CSV with one row per file, containing `fileName`, `takesUpSpace`, `isOriginalQuality`, `takenAt`, `sizeBytes`, and more.

---

### Step 2 тАФ Index your Google Drive

Run this once to create a local index of all photos/videos in your Drive:

```bash
cd /Users/hiijitesh/rclone_script

rclone lsjson -R --filter-from media_filter.txt "jiteshece:" > drive_index.json
```

- `media_filter.txt` restricts the scan to only photo/video extensions (jpg, png, heic, mp4, mov, etc.)
- This creates `drive_index.json` тАФ a map of every media file's path and modification date in your Drive
- This command is **read-only** тАФ it does not change anything in Drive

> тП▒я╕П Takes 1тАУ3 minutes depending on Drive size.

---

### Step 3 тАФ Generate the rclone backup commands

```bash
python3 backup_photos.py
```

The script will:
1. Read `metadata.csv` and filter rows where `takesUpSpace = true` AND `isOriginalQuality = true`
2. Load `drive_index.json` to build a filename lookup
3. Ask you for:
   - Your rclone remote name (e.g. `jiteshece:`)
   - The target backup folder name (e.g. `Photos_Backup`)
4. Match each filename and extract the year from `takenAt`
5. Generate `run_backup.sh` тАФ a bash script of `rclone copyto` commands

Example of what `run_backup.sh` looks like:
```bash
rclone copyto "jiteshece:Camera Uploads/IMG_1234.jpg" "jiteshece:Photos_Backup/2021/IMG_1234.jpg"
rclone copyto "jiteshece:Camera Uploads/VID_5678.mp4" "jiteshece:Photos_Backup/2019/VID_5678.mp4"
```

> тЬЕ **Nothing is copied yet.** Always review `run_backup.sh` before running it.

---

### Step 4 тАФ Run the backup

```bash
bash run_backup.sh
```

This copies every matched file **within Google Drive** (cloud-to-cloud) into year-wise subfolders. No data is downloaded to your computer.

## тЪая╕П Known Limitations

1. **Files must already be in Google Drive** тАФ This tool does not download from Google Photos. It assumes you have the same photos already uploaded to Google Drive (either manually or via old pre-2019 auto-sync).

2. **Filename collisions** тАФ If the same filename exists in multiple Drive folders, the script picks the first match and logs a warning in `run_backup.sh` as a comment. Check `run_backup.sh` for lines starting with `# WARNING`.

3. **Files not found** тАФ Any filenames from the CSV not found in Drive are saved to `missing_files.txt`. These you would need to handle separately (e.g. via Google Takeout).

4. **Year is based on `takenAt`** тАФ The folder year comes from the photo's capture date, not the upload date. If Google couldn't determine the capture date, the folder will be `Unknown_Year/`.

## ЁЯЫая╕П Troubleshooting

**`drive_index.json` is empty or very small**
тЖТ Check your rclone remote name: `rclone listremotes`

**Many files in `missing_files.txt`**
тЖТ Your Google Drive and Google Photos may not have the same files. Consider using Google Takeout to download missing originals.

**Python error: `metadata.csv not found`**
тЖТ Make sure `metadata.csv` is in the same directory as `backup_photos.py`

**`rclone copyto` fails in `run_backup.sh`**
тЖТ Run `rclone about jiteshece:` to check Drive connectivity and available space.
